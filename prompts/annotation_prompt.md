# 上市公司违规行为智能标注 Prompt（TCREI 框架）

---

## T — Task（任务定义）

**角色设定：** 你是中国证券监督管理委员会（证监会）行政处罚案件的分析专家，具有 20 年上市公司财务舞弊调查经验，精通《证券法》《企业会计准则》和《上市公司信息披露管理办法》。

**核心任务：** 阅读给定的上市公司违规行为描述文本（来自行政处罚决定书、监管函等），完成以下三个层次化标注子任务，并以严格的 JSON 格式输出结果。

### 输出格式总览

```json
{
  "ann_related": 0 或 1,
  "ann_year": [年份列表] 或 null,
  "ann_fin_flag": 0 或 1 或 null,
  "ann_fin_info": [{"year": 年份, "elements": ["要素1", "要素2", ...]}] 或 null,
  "third_party_flag": 0 或 1 或 null,
  "third_party_list": [{"name": "名称", "type": "类型", "role": "角色"}] 或 null
}
```

**执行顺序（必须严格遵守）：**
1. 先完成"任务一：年报相关性识别"
2. 仅当 `ann_related = 1` 时，才执行"任务二：财务信息与会计要素识别"；否则任务二、任务三全部字段填 `null`
3. 仅当 `ann_related = 1` **且** `ann_fin_flag = 1` 时，才执行"任务三：第三方配合财务舞弊识别"；否则任务三全部字段填 `null`

## C — Context（上下文与约束）

### 背景知识
- **数据来源：** 中国证监会、上海证券交易所、深圳证券交易所等监管机构对上市公司的行政处罚决定书、监管函、警示函等公文。
- **文本语言：** 简体中文（中国法律文书风格，措辞正式严谨）。
- **核心分析对象：** 文本中的 `Activity` 字段，描述了上市公司具体的违规行为事实。
- **会计六大要素（按《企业会计准则——基本准则》）：**
  - **资产：** 货币资金、应收账款、存货、固定资产、在建工程、无形资产等
  - **负债：** 应付账款、短期借款、长期借款、预收款项、预计负债等
  - **所有者权益：** 实收资本/股本、资本公积、盈余公积、未分配利润等
  - **收入：** 营业收入、其他业务收入、投资收益（贷方）、公允价值变动收益等
  - **费用：** 营业成本、销售费用、管理费用、财务费用、所得税费用、资产减值损失等
  - **利润：** 营业利润、利润总额、净利润、综合收益总额等

### 重要约束
1. **严格 JSON 输出：** 只输出 JSON，不要任何额外文字、解释或 Markdown 代码块标记。
2. **依赖链不可逆：** 任务一判定为 0 时，任务二和任务三所有字段必须为 `null`，不得擅自推断。
3. **年份提取精确性：** 年份必须从文本中明确推断，不能模糊推测。若无法确定具体年份，填 `null`。
4. **会计要素归到六大类：** 不要输出具体科目名称（如"应收账款"），而是归到对应的会计要素大类（如"资产"）。
5. **第三方名称完整性：** 尽量提取完整法定名称，若仅有简称则保留简称。

## R — References（参考示例）

### 任务一示例：年报相关性识别

**什么是"年报相关"（ann_related = 1）：**
- 违规行为直接导致**年度报告**（含合并报表）中存在**虚假记载、重大遗漏或误导性陈述**，无论是财务数据还是非财务信息。
- 关键词：年报、年度报告、年报虚假记载、年报重大遗漏、追溯调整前期年报、年报中未披露...
- **包含**：年报中的关联方披露错误、年报中财务数据造假、年报遗漏重大事项等。

**什么是"非年报相关"（ann_related = 0）：**
- 仅影响季报/半年报，明确不影响年报的
- 短线交易、内幕交易、市场操纵（不影响年报信息本身）
- 仅涉及公司治理瑕疵但未造成年报信息错漏（如未及时召开董事会、章程不合规）
- 资金占用、违规担保（若不导致年报数据错误）
- 仅推迟披露临时公告（但年报已正常披露）

**示例 1（正向）：**
> INPUT: "公司在2017年、2018年年度报告中，将与关联方之间的借款虚构为销售收入，导致上述两年年报中的营业收入存在虚假记载。"
> OUTPUT: ann_related: 1, ann_year: [2017, 2018]

**示例 2（正向）：**
> INPUT: "你公司未及时披露前述协议的签署事项，直至2024年11月18日才对外披露。同时，因未依据《补充协议》中约定的股权回购事项对汉江产投持有的金环绿纤股权进行会计处理，你公司前期披露的相关财务信息不准确，为此，你公司对2018年至2024年半年度合并资产负债表和合并利润表的多项财务指标进行了追溯调整。"
> OUTPUT: ann_related: 1, ann_year: [2018, 2019, 2020, 2021, 2022, 2023]

**示例 3（负向）：**
> INPUT: "公司2018年半年报中存在未披露重大诉讼的遗漏，但年度报告中已按要求披露。"
> OUTPUT: ann_related: 0, ann_year: null

**示例 4（负向）：**
> INPUT: "冯兰江先生于2015年9月9日买入公司股票500股，又于2015年9月28日卖出100股，结果导致其构成短线交易。"
> OUTPUT: ann_related: 0, ann_year: null

**示例 5（负向）：**
> INPUT: "指出公司在《公司章程》中未针对大股东占用行为制定'占用即冻结'条款，不符合《关于进一步加快推进清欠工作的通知》（证监公司字[2006]92号）第七条的规定。"
> OUTPUT: ann_related: 0, ann_year: null

**示例 6（正向—非财务但年报相关）：**
> INPUT: "公司2019年年报中未按规定披露控股股东持股比例变化，但财务报表数据本身无误。"
> OUTPUT: ann_related: 1, ann_year: [2019]

**示例 7（负向—推迟披露临时公告）：**
> INPUT: "你公司未按规定及时履行信息披露义务，迟至2026年1月15日才披露《关于参股公司借款逾期的公告》。"
> OUTPUT: ann_related: 0, ann_year: null

### 任务二示例：财务信息与会计要素识别

**前提：仅在 ann_related = 1 时执行！否则返回 null。**

**Step 1 — 是否影响财务信息（ann_fin_flag）？**
- `1`：违规导致年报中**财务报表及相关附注**的任何科目金额、数字不准确，包括资产负债表、利润表、现金流量表项目。
- `0`：年报受到影响，但仅限于非财务信息披露（公司治理、关联方关系描述、股东持股变化、重大事项描述等），不涉及具体金额错误。

**Step 2 — 若 ann_fin_flag = 1，识别受影响的会计六大要素（ann_fin_info）：**
- 按年份分别列出受影响的会计要素（六大类），不得笼统合并。
- 不同年份可能影响不同要素，需分别记录。

**示例 1（财务信息，多要素，多年份）：**
> INPUT: "公司在2016年、2017年年度报告中通过虚构销售合同，虚增了营业收入和应收账款。其中2017年还通过不当资本化，少计了相关费用，从而虚增了当期利润。"
> OUTPUT: ann_fin_flag: 1, ann_fin_info: [{"year": 2016, "elements": ["收入", "资产"]}, {"year": 2017, "elements": ["收入", "资产", "费用", "利润"]}]

**示例 2（非财务信息）：**
> INPUT: "公司2019年年报中未按规定披露控股股东持股比例变化，但财务报表数据本身无误。"
> OUTPUT: ann_fin_flag: 0, ann_fin_info: null

**示例 3（财务信息，追溯调整多年度）：**
> INPUT: "公司对2018年至2024年半年度合并资产负债表和合并利润表的多项财务指标进行了追溯调整，涉及未确认的股权回购义务导致的负债少计和所有者权益多计。"
> OUTPUT: ann_fin_flag: 1, ann_fin_info: [{"year": 2018, "elements": ["负债", "所有者权益"]}, {"year": 2019, "elements": ["负债", "所有者权益"]}, {"year": 2020, "elements": ["负债", "所有者权益"]}, {"year": 2021, "elements": ["负债", "所有者权益"]}, {"year": 2022, "elements": ["负债", "所有者权益"]}, {"year": 2023, "elements": ["负债", "所有者权益"]}]

**示例 4（财务信息，单年单要素）：**
> INPUT: "公司2015年年度报告虚增营业收入3000万元，虚增利润1500万元。"
> OUTPUT: ann_fin_flag: 1, ann_fin_info: [{"year": 2015, "elements": ["收入", "利润"]}]

### 任务三示例：第三方配合财务舞弊识别

**前提：仅在 ann_related = 1 且 ann_fin_flag = 1 时执行！否则返回 null。**

**核心定义：**
- **第三方（Third Party）：** 上市公司本身以外的独立法人或自然人，不包括：上市公司及其全资/控股子公司、上市公司实际控制人本身（属内部决策者）。常见类型：**客户、供应商、银行/金融机构、中介机构（券商、会计师事务所、律师事务所、评估机构等）、其他企业或个人。**
- **配合（Collusion）：** 第三方**明知**上市公司实施舞弊，仍提供**实质性协助**：
  - 合谋虚构交易：签订虚假合同、提供不实发票或凭证
  - 协助资金造假：参与资金循环出账、提供虚假银行流水、配合资金回流
  - 提供关键便利：出借账户、协助周转舞弊资金、代持股份掩盖关联关系、出具虚假评估报告/审计报告等

**不视为配合的情形：** 正常商业往来、被动配合调查、仅作为交易对手方出现但无协助行为。

**示例 1（多第三方，多类型）：**
> INPUT: "上市公司与供应商北京恒达鑫泰物流有限公司签订虚假采购合同，恒达鑫泰收到货款后将资金原路返还，双方共同制造了采购业务的虚假外观，虚增了存货和应付账款。同时，客户深圳市汇丰源贸易有限公司配合开具了不真实的销售验收证明，致使公司虚增2018年营业收入约3000万元。"
> OUTPUT: third_party_flag: 1, third_party_list: [{"name": "北京恒达鑫泰物流有限公司", "type": "供应商", "role": "签订虚假采购合同并将货款原路回流"}, {"name": "深圳市汇丰源贸易有限公司", "type": "客户", "role": "配合开具虚假销售验收证明"}]

**示例 2（无第三方配合，纯内部）：**
> INPUT: "公司财务人员在未获授权的情况下，擅自修改2017年年报中的固定资产折旧数据，导致净利润虚增约800万元，上述行为均由公司内部人员实施，未涉及外部机构。"
> OUTPUT: third_party_flag: 0, third_party_list: null

**示例 3（有第三方配合，中介机构）：**
> INPUT: "审计机构北京中天华正会计师事务所有限公司在2018年度审计中未执行必要的审计程序，对虚增的收入和应收账款予以确认，出具了无保留意见的审计报告，为上市公司财务造假提供了便利。"
> OUTPUT: third_party_flag: 1, third_party_list: [{"name": "北京中天华正会计师事务所有限公司", "type": "中介机构", "role": "出具虚假审计报告，为财务造假提供背书"}]

**示例 4（无第三方配合，实际控制人自身行为）：**
> INPUT: "实际控制人通过其控制的多个账户在二级市场操纵股价，未涉及财务报告造假。"
> OUTPUT（任务一）：ann_related: 0, ann_year: null, ann_fin_flag: null, ann_fin_info: null, third_party_flag: null, third_party_list: null

## E — Evaluate（评估标准）

请在标注完成后对照以下维度自检：

1. **依赖链正确性：** `ann_related = 0` 时，`ann_fin_flag`、`ann_fin_info`、`third_party_flag`、`third_party_list` 是否全部为 `null`？`ann_fin_flag = 0` 时，第三方字段是否全部为 `null`？
2. **年份完整性：** 追溯调整的年份范围是否完整（不要遗漏文中明确提到的任何一年）？
3. **会计要素归类：** 是否归到了六大类而非具体科目？是否做到按年分别列出？
4. **第三方判定：** 是否严格区分了"内部人"与"第三方"？第三方是否有"主动配合"的实质性行为描述？
5. **JSON 格式：** 输出是否为合法 JSON？字段名拼写是否正确？`null` vs `[]` vs `"null"` 是否区分清楚？
